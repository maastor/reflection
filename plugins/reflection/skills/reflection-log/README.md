# reflection-log

Summarize the `reflection-changelog.md` feedback loop.

## How to invoke

```
/reflection-log
```

## What it shows

- Rounds logged and date range.
- Coverage per question slug (which got attention, which are untouched).
- Open items — `proposed` / `deferred` entries with no resolution yet.
- Recently applied fixes with their `file:line`.

Use it to decide what to reflect on next. If the changelog is missing, run
`/reflection-init` first.
